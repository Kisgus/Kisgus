## Current Work

### SAIBA

SAIBA is a privacy-first operating layer for expert teams that need agents to
help without turning customer work into a black box.

The loop is deliberately simple:

```text
customer signal -> shared memory -> human decision -> scoped agent work -> receipt
```

That means discovery calls, Slack handovers, CRM notes, GitHub issues, delivery
reviews, and customer objections become structured context. Agents can help with
the work, but risky artifacts stay behind human approval.

### Link2Post and Public Receipts

The public content system is built around source-tethered posts: real releases,
PRs, demos, and customer-safe receipts become drafts for the right person or
company account. The point is not to flood channels. The point is to make real
work easier to understand from the outside.

### Agent Workbench

The technical workbench is MCP-first, testable, and receipt-oriented. I care
about systems where the agent can explain what it saw, what it changed, why it
changed it, and which human approved the public edge.

## AI x Crypto and Digital Assets

The broader thread in my work is how AI, crypto, and digital assets become
useful inside institutions without losing privacy, agency, or trust.

I have worked across open-web products, protocol ecosystems, public-sector
blockchain research, AI x crypto intelligence, and digital asset investment
communities. That includes NEARWEEK, Linkdrop, Trustlines, brainbot/Raiden,
community currencies, public-sector research, and advisory work around

My current focus combines four lanes:

- public-sector and institutional governance around AI x crypto.
